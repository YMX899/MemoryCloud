# 开源广场记忆与 Skill 风险标注

本文记录从 GitHub 等公开来源转换 Memory Suite 时的风险边界。公开广场允许政治人物、争议营销、人格蒸馏和数字生命类内容，但必须以“方法记忆 / 风险审查 / 工作流记忆”发布，不得包装为本人、授权代表、法律身份或事实担保。

## 总原则

- 公开包只提供上下文、方法、审查清单和行为倾向，不能覆盖系统策略、当前用户指令、安全规则或专业合规要求。
- 不直接复制长篇 `SKILL.md`、访谈、书籍、训练材料或社交媒体内容；只发布摘要化、结构化、可追溯的工作记忆。
- `provenance.source_urls` 必须保留 GitHub 仓库或公开来源链接；许可证缺失时不得按可再授权内容搬运，只能做公开来源事实级摘要或先不上架。
- 涉及真实人物时必须声明：不是本人、不是授权代表、不是私密记忆；只能作为公开方法论蒸馏。
- 涉及用户、亲友、同事、聊天记录、照片、社交账号等个人资料时，默认私有处理；公开包只能发布采集流程和脱敏规范，不发布样本个人数据。

| 玄学人格 | 八字人格等 | 被误当作医疗、法律、投资或人生决定依据 | 标注娱乐/文化视角；高风险决策必须转向专业意见和现实证据 |
| Skill 治理与评测 | skill-fidelity-bench 等 | 被用于对抗审计、规避边界 | 发布为防御性质量审查流程；不得提供绕过安全边界的操作 |

## 本批处理原则

- GitHub MIT 授权仓库可转为 Memory Suite，但仍只做摘要化转换。
- 无许可证或许可证不明的仓库暂不上架；如必须引用，只能在风险文档中列为“待授权/待审查来源”。
- 政治人物和争议营销内容允许上架，但默认标签应包含 `risk-review`、`public-methodology` 或类似边界词。
- 每个包的 `MEMORY.md` 必须包含安全边界、使用场景、核心工作记忆、Agent 使用方式和检索触发词。
- 每个包的 `DREAMS.md` 必须包含反思性边界，提醒不要把风格模仿当成判断质量。

## 待人工复核

- 真实人物授权声明是否足够醒目。
- 是否包含可被理解为投资、医疗、法律、选举动员、金融营销或人格冒充的高风险建议。
- 是否存在长段原文搬运、歌词/书籍/付费材料摘录或未授权隐私样本。
- 是否需要将 `visibility` 从 `public` 降为 `unlisted` 或 `private`。
